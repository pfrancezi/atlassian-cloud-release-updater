# Fields Configuration Guide

Create the fields below in the Jira project where release issues will be created.

The scripts skip optional custom fields when the corresponding `CF_*` variable is left empty.

## Required Jira basics

| Purpose | Jira field | Type | Required? | Notes |
|---|---|---:|---:|---|
| Project | Native field | Project | Yes | Provided at runtime as `JIRA_PROJECT_KEY`. |
| Issue type | Native field | Issue type | Yes | Provide name or ID at runtime as `JIRA_ISSUE_TYPE`. |
| Summary | Native field | Text | Yes | Generated by the script. |
| Description | Native field | ADF | Yes | Generated by the script. |

## Recommended custom fields

| Script variable | Suggested name | Type | Required? | Expected values / behavior |
|---|---|---:|---:|---|
| `CF_ADMIN_ID` | Atlassian Change ID | Short text | Strongly recommended | Stores the opaque Admin/Community change ID. Used for matching. |
| `CF_SOURCE` | Source | Multi-select | Strongly recommended | `Private`, `Community`. |
| `CF_PRODUCTS` | Products | Multi-select | Recommended | Technical product tags, for example `jira-software`, `jira-servicedesk`, `confluence`, `cloud-admin`. |
| `CF_CHANGE_STATUS` | Change Status | Single-select | Recommended when using the Private source | Private/Admin lifecycle status. Examples: `NEW THIS WEEK`, `ROLLING_OUT`, `COMING_SOON`, `GENERALLY_AVAILABLE`, `PLANNED`, `EXPERIMENT`, `DEPRECATED`, `ANNOUNCEMENT`. |
| `CF_COMMUNITY_STATUS` | Community Status | Single-select | Recommended | Community lifecycle status. It stores either `NEW THIS WEEK` during first appearance or the latest status returned by Community, for example `ROLLING_OUT`, `COMING_SOON`, `GENERALLY_AVAILABLE`. |
| `CF_COMMUNITY_CHANGE_TYPE` | Community Change Type | Multi-select | Recommended | Community change type values returned by the API. |
| `CF_COMMUNITY_FIRST_PUBLISHED` | Community First Published | Date / JPD date range | Recommended | First public publication date from Community. |
| `CF_COMMUNITY_WEEK_START` | Community Week Start | Date / JPD date range | Recommended | Monday of the first-published week. |
| `CF_COMMUNITY_WEEK_END` | Community Week End | Date / JPD date range | Recommended | Sunday of the first-published week. |

## Optional custom fields

| Script variable | Suggested name | Type | Expected values / behavior |
|---|---|---:|---|
| `CF_CHANGEKEY` | Change Key | Short text | Optional public/provider key when available. |
| `CF_CHANGE_TYPE` | Change Type | Single-select | Private/Admin change type. |
| `CF_SITE_STATUS` | Site Release Status | Single-select | Private/Admin site release status, for example `ALL` or `SOME`. |
| `CF_POSTDATE` | Post Date | Date / JPD date range | Private/Admin post date. |
| `CF_RELEASE_START` | Release Start | Date / JPD date range | Private/Admin release start date. |
| `CF_RELEASE_END` | Release End | Date / JPD date range | Private/Admin release end date. |
| `CF_IS_DELAYABLE` | Is Delayable | Multi-select | Values: `Yes`, `No`. |
| `CF_IS_DELAYED` | Is Delayed | Multi-select | Values: `Yes`, `No`. |
| `CF_COMMUNITY_RELEASE_START` | Community Release Start | Date / JPD date range | Community release start date. |


## Date field difference

Use one script or the other, not both for the same project:

| Project type | Script | Date payload format |
|---|---|---|
| Jira Product Discovery | `weekly_release_updater_jpd.py` | JSON string: `{"start":"2026-06-22","end":"2026-06-22"}` |
| Jira Software | `weekly_release_updater_jira_software.py` | Plain date string: `2026-06-22` |

## How to find custom field IDs

Open Jira field configuration or use the Jira REST API:

```bash
curl -u email@example.com:API_TOKEN \
  -H "Accept: application/json" \
  "https://your-site.atlassian.net/rest/api/3/field"
```

Look for the field name and copy its `id`, for example `customfield_13005`.

Then replace the constants near the top of the script:

```python
CF_SOURCE = "customfield_13005"
CF_ADMIN_ID = "customfield_13214"
CF_COMMUNITY_STATUS = "customfield_13576"
```

## Option creation

When `AUTO_CREATE_FIELD_OPTIONS = True`, the script can create missing options for configured select and multi-select fields. This includes new Community status values that may appear over time.

Create or allow at least these values:

| Field | Common values |
|---|---|
| Source | `Private`, `Community` |
| Change Status | `NEW THIS WEEK`, plus statuses returned by the Private Admin source. |
| Community Status | `NEW THIS WEEK`, plus statuses returned by the Community API. This field should be single-select. |
| Is Delayable / Is Delayed | `Yes`, `No` |
